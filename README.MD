Конспект: DOM-события в JavaScript

1. Что такое события и как с ними работать

Событие — сигнал от браузера о том, что что-то произошло: действие пользователя (клик, ввод, прокрутка) или завершение процесса (загрузка страницы).
JavaScript позволяет «слушать» эти сигналы и реагировать на них пользовательским кодом.

Главный инструмент — addEventListener. Он вешается на DOM-элемент и принимает два аргумента: имя события и колбэк-функцию.

```javascript
const btn = document.querySelector('button');
btn.addEventListener('click', () => console.log('Клик!'));
```

---

2. Объект события

Когда событие происходит, браузер автоматически создаёт объект Event и передаёт его в обработчик первым аргументом. В нём хранится вся информация о событии: тип, целевой элемент, координаты, нажатая клавиша и т.д.

```javascript
btn.addEventListener('click', (event) => {
  console.log(event.type); // 'click'
});
```

Ключевые свойства

· event.target — элемент, на котором непосредственно произошло событие (вложенный элемент при клике внутри родителя).
· event.currentTarget — элемент, на котором висит обработчик (всегда тот, на ком вызван addEventListener).

---

3. Популярные типы событий

Мышь

· click — одиночный клик.
· mouseenter / mouseleave — курсор входит в границы элемента / покидает их (не всплывают, в отличие от mouseover/mouseout).

Клавиатура

· keydown — клавиша нажата (срабатывает и для служебных клавиш). Получить символ: event.key.

Формы и поля ввода

· input — срабатывает при каждом изменении значения (ввод, вставка, удаление).
· change — после завершения изменения и потери фокуса (удобно для чекбоксов, селектов, range).
· submit — отправка формы. Обычно требуется event.preventDefault(), чтобы остановить перезагрузку страницы.
· focus / blur — получение / потеря фокуса элементом.

Документ и окно

· DOMContentLoaded — HTML полностью загружен и разобран, DOM готов (скрипт можно выполнять до загрузки картинок).
· scroll — прокрутка страницы. Для отслеживания позиции используй window.scrollY.

---

4. Управление поведением по умолчанию

У многих событий есть встроенное поведение браузера (переход по ссылке, отправка формы, выделение текста). Чтобы его отменить, вызывают метод объекта события:

```javascript
event.preventDefault();
```

---

5. Всплытие (Bubbling) и его остановка

Когда событие происходит на вложенном элементе, оно по цепочке поднимается вверх по родителям. Это и есть всплытие.

· event.stopPropagation() — полностью останавливает всплытие (выше событие не пойдёт).

Всплытие — не баг, а фундаментальная механика, на которой строится делегирование событий.

---

6. Делегирование событий

Идея: вместо того чтобы вешать обработчик на каждый дочерний элемент, мы вешаем один обработчик на общего родителя и внутри анализируем event.target.

```javascript
list.addEventListener('click', (event) => {
  if (event.target.tagName === 'LI') {
    console.log(event.target.textContent);
  }
});
```

Вспомогательные методы

· event.target.closest(селектор) — ищет ближайшего родителя (или сам элемент), подходящего под CSS-селектор. Удобно, чтобы найти карточку по кнопке внутри неё.
· event.target.matches(селектор) — проверяет, соответствует ли сам target селектору (полезно для динамических элементов).

Работа с динамическими элементами
Если кнопки создаются уже после загрузки страницы, делегирование спасает: обработчик на родителе (или на document) сработает и для них.

---

7. Пользовательские данные через data-атрибуты

Часто нужно привязать к элементу идентификатор или другие данные. В HTML используют data-* атрибуты:

```html
<button data-id="15" data-action="delete">Удалить</button>
```

В JavaScript доступ к ним через element.dataset:

```javascript
console.log(button.dataset.id);     // "15"
```

---

8. Особенности this в обработчиках

· В обработчике, объявленном как обычная функция, this указывает на элемент, на котором висит обработчик (event.currentTarget).
· В стрелочной функции своего this нет, поэтому для получения текущего элемента надёжнее использовать event.currentTarget.

```javascript
button.addEventListener('click', function() {
  console.log(this);              // button
});
button.addEventListener('click', (e) => {
  console.log(e.currentTarget);   // button
});
```

9. Продвинутые приёмы

Debounce (для событий ввода)
При частых событиях (input, scroll, resize) вызов функции можно «притормозить», чтобы не нагружать браузер. Простая реализация debounce:

```javascript
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

input.addEventListener('input', debounce(() => {
  console.log('Запрос поиска');
}, 300));
```

Асинхронные обработчики
Внутри обработчика можно использовать async/await. Браузер не будет ждать завершения промиса, но это удобно для запросов к серверу:

```javascript
button.addEventListener('click', async () => {
  const response = await fetch('/users');
  const data = await response.json();
});
```

Удаление обработчика
Чтобы перестать слушать событие, используют removeEventListener. Важно: нужно передать ту же самую функцию (не анонимную):

```javascript
const handler = () => { /* ... */ };
button.addEventListener('click', handler);
button.removeEventListener('click', handler);
```

Валидация форм перед отправкой
При обработке submit проверяем поля вручную и вызываем preventDefault(), если есть ошибки:

```javascript
form.addEventListener('submit', (event) => {
  if (input.value.trim() === '') {
    event.preventDefault();
    alert('Поле не может быть пустым');
  }
});
```
